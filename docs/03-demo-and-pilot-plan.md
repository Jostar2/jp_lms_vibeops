# Demo And Pilot Plan

## Demo Rule

No live autonomous demo for executive presentation.

Use three synchronized assets:

1. Pre-recorded walkthrough.
2. Clickable prototype.
3. PNG fallback deck.

## Demo A: Closed Loop Learning Intervention

Purpose: prove the core product.

Flow:

1. Student struggles at a lecture segment.
2. AI explains the likely prerequisite gap.
3. Instructor sees a decision queue item.
4. Co-Creation Studio drafts a safer explanation and checkpoint.
5. Instructor approves release.
6. Two weeks later, Impact Ledger shows effect and uncertainty.

Required proof:

- no individual risk list
- evidence card includes model, uncertainty, action, measurement
- instructor approval is explicit
- post-action measurement is visible

## Demo B: Japanese University Fit

Purpose: prove this is not a generic Western LMS AI.

Flow:

1. Japanese academic calendar and course registration terms are first-class fields.
2. Student support copy changes by Honne/Tatemae context and Keigo level.
3. Meiwaku feedback lets a user challenge or withdraw an AI judgement.
4. APPI data access reason is visible before sensitive views open.

Required proof:

- native Japanese copy review status
- no stigmatizing learner-facing risk labels
- consent and purpose limitation are explicit

## Demo C: Pilot Readiness For NetLearning

Purpose: turn presentation interest into a follow-up decision.

Flow:

1. Integration fit page shows LTI/OneRoster/SSO/CSV options.
2. Pilot packet shows target university criteria.
3. Measurement protocol shows outcome, method, sample, and report cadence.
4. Term sheet draft shows roles, IP, data controller/processor, and support model.

Required proof:

- clear ask to NetLearning
- 90-day pilot plan
- APPI/IRB readiness checklist
- fallback integration path

## 90-Day Pilot MLP

| Period | Output |
| --- | --- |
| Days 0-14 | Decision Brief, integration fit, APPI checklist, pilot candidate shortlist |
| Days 15-30 | Clickable demo, Japanese copy review, data mapping, measurement preregistration |
| Days 31-60 | Limited pilot connector, instructor workflow, event ledger, xAI cards |
| Days 61-90 | Intervention measurement, monthly report, go/no-go review |

## Pilot Success Metrics

- Instructor approval completion rate.
- Student perceived support without surveillance.
- Intervention uplift with confidence interval.
- PPV and calibration for targeted support signals.
- Reduction in repeated struggle segment rewatch or failed checkpoint.
- Legal/IT approval cycle time.
- NetLearning integration effort estimate.
