# 90-Day Measurement Protocol

## Pilot Objective

Prove one measurable AI learning operations loop without production-risk automation.

## Timeline

| Period | Work |
| --- | --- |
| Days 0-14 | course selection, data mapping, APPI/IRB review, baseline fixture |
| Days 15-30 | adapter mock, xAI card review, instructor approval workflow |
| Days 31-60 | intervention execution in controlled course context |
| Days 61-75 | two-week effect measurement and limitation review |
| Days 76-90 | impact report, go/no-go, integration estimate |

## Metrics

| Metric | Type | Source |
| --- | --- | --- |
| segment rewatch rate delta | primary | LMS video activity |
| checkpoint correct-rate delta | secondary | LMS quiz/checkpoint |
| instructor approval completion | operational | approval ledger |
| student perceived support | safety | survey |
| perceived surveillance | safety | survey |
| rollback count | safety | approval/rollback ledger |

## Publication Rule

Only aggregate pilot results can be shown externally.

Every effect claim requires:

- method
- baseline
- observed value
- uncertainty or limitation note
- sample boundary
- decision: retain, revise, rollback, or insufficient evidence

## Go/No-Go Criteria

Go:

- outcome moves in expected direction
- instructor workflow is accepted
- no unresolved privacy blocker
- integration effort remains bounded

No-go or revise:

- measurement is inconclusive
- APPI/IRB review blocks planned data use
- student feedback indicates surveillance concern
- instructor approval workflow is too slow
