# Term Sheet Outline

This is a commercial and operating outline, not a legal contract.

## Parties

- NetLearning or platform-side partner.
- Candidate university or department.
- JP LMS VibeOps builder/operator.
- Legal/privacy reviewer where applicable.

## Pilot Scope

- One course or department context.
- One S01 closed-loop intervention.
- Aggregate or pseudonymous learning activity data by default.
- No autonomous publishing without instructor approval.

## Responsibilities

| Area | Owner |
| --- | --- |
| Course selection | university |
| Data export permission | university |
| Platform context | NetLearning |
| Control plane and evidence UX | JP LMS VibeOps |
| APPI/IRB review | university + legal/privacy reviewer |
| Measurement report | JP LMS VibeOps + course owner |

## IP And Data

Open items:

- ownership of pilot-specific content improvements
- model output reuse
- event data retention
- deletion timeline
- publication permission for aggregate results

## Exit

At day 90, decide:

- continue to integration pilot
- revise and rerun
- stop and delete pilot data
