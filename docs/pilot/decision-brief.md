# Decision Brief

## Ask

Approve a 90-day JP LMS VibeOps pilot preparation track with NetLearning and one candidate Japanese university department.

The requested decision is not production deployment. The decision is permission to prepare a constrained pilot package, integration path, data review, and measurement protocol.

## What JP LMS VibeOps Proves

JP LMS VibeOps proves whether an AI learning operations layer can improve a course workflow without replacing the existing LMS.

The pilot centers on one closed loop:

```text
lecture struggle signal -> instructor xAI card -> approved content improvement -> two-week measurement -> impact ledger
```

## Why Now

Generic AI LMS demos are easy to imitate. A measured, approval-gated, APPI-aware learning operations loop is harder to imitate and more relevant to university buyers.

## Decision Items

| Item | Decision Needed |
| --- | --- |
| Pilot owner | Name NetLearning-side and university-side accountable owners |
| Candidate course | Select one course where aggregated learning activity data is available |
| Integration path | Choose LTI/OneRoster/API/CSV fallback for pilot preparation |
| Data review | Start APPI/privacy and IRB/pre-registration review |
| Measurement | Approve 90-day measurement protocol draft |

## Non-Decision

This brief does not request:

- production deployment
- autonomous content publishing
- identified student risk lists
- grade-changing automation
- legal approval without counsel review
