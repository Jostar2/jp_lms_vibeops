# Codex Full IA Refinement Plan

## 전제

숨긴 메뉴를 전부 노출하는 방향이 맞다. 지금 단계에서 걷어내면 "AI LMS가 실제 운영 전체를 이해한다"는 인상이 약해진다. 다만 모든 메뉴가 보이는 순간 위험도 커진다. 각 화면이 그냥 기능 목록처럼 보이면 일반 LMS + AI 배지로 후퇴한다.

따라서 전체 IA는 다음 원칙으로 정교화한다.

```text
모든 메뉴는 하나의 운영 질문에 답해야 한다.
모든 AI는 VIEW / DECIDE / GENERATE 중 하나의 사용자 행동으로 읽혀야 한다.
모든 AI는 Detect / Prioritize / Recommend / Draft / Verify 중 하나의 역할을 가져야 한다.
모든 화면은 "왜 이 화면이 필요한가"를 첫 화면에서 증명해야 한다.
```

## 전체 구조

### Student IA

| 메뉴 | 운영 질문 | AI Intent | AI Role | 뾰족하게 만들 지점 |
|---|---|---:|---:|---|
| 대시보드 | 오늘 무엇부터 해야 완주 가능한가 | DECIDE | Prioritize | 오늘 3개만 남기는 판단 근거와 대안 경로 |
| 오늘의 학습 | 오늘 계획을 시간 블록으로 어떻게 실행할까 | DECIDE | Recommend | 집중 지속시간, 마감, 부담도를 함께 본 재계획 |
| 강의 탐색 | 다음 강의를 왜 추천받았는가 | VIEW | Prioritize | 추천 가중치와 학생이 직접 조정하는 선호 슬라이더 |
| 강의 수강 | 지금 어디서 막히고 무엇을 확인해야 하나 | VIEW | Detect | 영상 구간별 막힘 신호, 메타인지 체크, 즉시 확인 문제 |
| 과제 · 토론 | AI 도움을 어디까지 써도 되는가 | DECIDE | Verify | 힌트 단계, 사용 로그, 정답 생성 차단, 제출 전 투명성 |
| 피드백 | 무엇을 고치면 다음 점수가 올라가는가 | VIEW | Recommend | 교수 피드백과 AI 보완 분석을 분리해서 보여주기 |
| 학습 동반자 | 지금 나에게 맞는 설명/질문 방식은 무엇인가 | GENERATE | Recommend | 소크라틱 질문, 힌트 단계 제한, 출처 기반 답변 |
| 나의 인사이트 | 내 학습 습관에서 장기적으로 바꿔야 할 것은 무엇인가 | VIEW | Detect | 이해도보다 행동 패턴: 복귀율, 확인 문제 회피, 과부하 |

## 정교화 방향

### 1. 메뉴가 많아져도 각 화면의 첫 문장은 달라야 한다

현재 일부 화면은 "AI가 도와준다"는 문법이 반복된다. 앞으로는 각 화면의 첫 문장이 운영 질문이 되어야 한다.

- 나쁜 예: "AI가 학습을 도와줍니다."
- 좋은 예: "오늘 112분 안에 완주하려면 무엇을 포기해야 하나?"
- 좋은 예: "AI가 확신하지 못한 8건만 교수가 먼저 봅니다."
- 좋은 예: "이 초안은 교수자의 말투를 어떻게 반영했나?"

### 2. 화면별 대표 AI object를 하나로 제한한다

메뉴가 많을수록 화면 안의 AI 객체는 줄여야 한다. 각 화면은 하나의 대표 object를 가져야 한다.

- 대시보드: `Decision Queue`
- 오늘의 학습: `Pace Plan`
- 강의 탐색: `Recommendation Receipt`
- 강의 수강: `Struggle Segment`
- 과제 · 토론: `AI Use Boundary`
- 피드백: `Improvement Delta`
- 학습 동반자: `Socratic Turn`
- 나의 인사이트: `Learner Pattern`
- 티칭 홈: `Teaching Decision Queue`
- 학습 설계: `Course Blueprint`
- 루브릭 빌더: `Rubric Contract`
- Co-Creation: `Draft Canvas`
- AI 평가 지원: `Uncertainty Queue`
- 클래스 건강도: `Cohort Pattern`
- 학습 개입: `Intervention Draft`
- 학기 인사이트: `Impact Ledger`

### 3. NetLearning이 강하게 볼 지점

NetLearning은 "기능이 많다"보다 "운영자가 실제로 겪는 판단 비용을 줄인다"에 반응할 가능성이 크다.

강하게 보여야 할 장면:

- AI가 채점 24건을 했다는 것보다, `교수가 봐야 할 불확실 8건만 남겼다`.
- AI가 콘텐츠를 생성했다는 것보다, `난관 구간을 감지하고 교수자 말투로 보완자료를 만든 뒤 2주 후 효과를 측정한다`.
- AI가 학생에게 추천했다는 것보다, `오늘의 부담 리스크와 완주 가능성을 함께 보고 계획을 줄인다`.
- AI가 학생을 감시한다는 느낌보다, `개인명보다 반 단위 패턴과 지원 행동을 먼저 보여준다`.

### 4. 화면별 삭제는 나중에 하되, 의미 중복은 지금부터 줄인다

메뉴는 전부 노출한다. 대신 각 화면 안에서 반복되는 요소는 줄인다.

- Dashboard마다 Companion 전체 채팅을 반복하지 않는다.
- 모든 카드에 AI 배지를 붙이지 않는다.
- `요약/자막/챗봇`처럼 흔한 AI 기능은 전면 주인공으로 세우지 않는다.
- 핵심은 항상 판단, 근거, 승인, 측정이다.

## 다음 패치 제안

1. 모든 노출 메뉴를 클릭했을 때 첫 화면에 `운영 질문`이 보이도록 헤드라인을 조정한다.
2. 각 화면에 `aiMeta()`를 1-2개만 배치해 Intent/Role을 표준화한다.
3. 숨겨져 있던 12개 화면 중 우선 `오늘의 학습`, `과제 · 토론`, `루브릭 빌더`, `클래스 건강도`, `학습 개입`을 정교화한다.
4. 각 화면별 대표 AI object 이름을 UI에 직접 노출한다.
5. 이후 NetLearning 발표 흐름에서 실제로 약한 화면을 걷어낸다.

## 결론

전부 노출은 맞다. 다만 "모든 메뉴가 있다"가 목표가 아니라 "교육 운영 전체가 하나의 AI 판단 체계로 연결되어 있다"가 목표다.

따라서 다음 단계는 삭제가 아니라 각 메뉴를 더 뾰족하게 만드는 것이다. 특히 숨겨져 있던 화면들은 범용 기능 화면이 아니라, 운영 질문에 답하는 화면으로 재정의해야 한다.
